# GEO 인용률 개선 가이드 — 스키마마크업 이외의 방법

작성일: 2026-07-08. 이 문서는 2024–2026 공식 논문/플랫폼 공식 문서/대규모 산업 데이터를 근거 수준별로 정리하고, 각 항목을 이 저장소에서 어떻게 개발로 연결할 수 있는지 매핑한 가이드입니다.

근거 표기: **[논문]** 동료심사 연구 · **[공식]** 엔진 제공사 1차 문서 · **[산업]** 대규모 산업 데이터(인과 검증 없음)

## 0. 전제: 스키마마크업의 실제 위치

- **[산업]** Ahrefs의 인과 실험(JSON-LD를 새로 추가한 1,885개 페이지 vs 대조군 4,000개, 2025-08~2026-03)에서 JSON-LD 추가만으로는 AI Overviews/ChatGPT 인용 증가가 확인되지 않았습니다. LLM은 검색 시점에 **보이는 본문 텍스트**를 읽습니다.
- **[공식]** 반면 Bing/Copilot은 스키마를 LLM 이해에 활용한다고 명시했고, ChatGPT 쇼핑은 구조화 메타데이터(피드+Product 스키마)로 상품을 선정합니다.
- 결론: 스키마는 **쇼핑 파이프라인과 Bing 대상 채널**로서 유지하되, 인용률의 본체는 아래 항목들이 좌우합니다. 스키마 내용과 보이는 본문의 일치(visible-text parity)가 전제 조건입니다.

## 1. 접근성 — 봇이 못 읽으면 인용도 없다 (우선순위 1)

- **[공식]** 답변 노출용 봇과 학습용 봇을 구분해 robots.txt/CDN/WAF를 점검하세요.
  - 답변 노출: `OAI-SearchBot`(ChatGPT 검색), `PerplexityBot`, `Googlebot`(AI Overviews/AI Mode) — **차단하면 답변에서 제외**
  - 학습 전용: `GPTBot`, `Google-Extended`, `CCBot` — 차단해도 답변 노출과 무관 (`Google-Extended` 차단은 AI Overviews에 영향 없음)
- **[산업]** Cloudflare 등 CDN이 관리형 AI 봇 차단을 기본 제공하기 시작해, 콘텐츠 문제로 오인하기 전에 인프라부터 확인해야 합니다.

**개발 적용**: PDP 도메인의 robots.txt와 CDN 설정을 감사하는 체크 스크립트. 서버 로그에서 위 봇들의 크롤 히트를 집계하면 "인용의 선행 지표"가 됩니다 (docs/geo-citation-measurement-guide.md 1-3절 참고).

## 2. 상품 피드 — 상품 질의에서 가장 레버리지가 큰 채널

- **[공식]** ChatGPT 쇼핑은 광고가 아닌 오가닉 선정이며, 근거는 구조화 메타데이터입니다. OpenAI 상품 피드 스펙(JSONL/CSV/TSV/Parquet, HTTPS 푸시, 15분 주기 갱신 가능)에 직접 피드를 제출하거나, Shopify 카탈로그/Google Merchant Center 경유가 가능합니다.
- **[공식]** 가격·재고·변형(variant) 데이터의 **신선도**가 선정 요인입니다. Bing은 IndexNow로 갱신 신호를 받습니다.

**개발 적용**: `pdp-geo-generator-agent`의 정규화된 product signal을 OpenAI 피드 스펙 JSONL로 직렬화하는 export 모듈(`feed-exporter`)을 추가하면, 기존 파이프라인 산출물을 그대로 쇼핑 채널에 연결할 수 있습니다. 스키마 Product/Offer와 피드 값의 일치 검증은 기존 validate 단계에 추가 가능합니다.

## 3. 콘텐츠 전술 — 통계·인용문·출처 (원 GEO 논문의 상위 전술)

- **[논문]** GEO 논문(KDD 2024): 통계 추가, 신뢰 가능한 인용문 추가, 출처 인용이 상위 성과(가시성 최대 ~40%, 하위 랭크 소스는 최대 +115%). **키워드 스터핑은 역효과(-10%)**.
- **[논문]** C-SEO Bench(NeurIPS 2025)의 교정: 문구 주입 트릭 다수는 무효/역효과이며, **검색 관련성과 컨텍스트 내 위치가 지배적**. 같은 전술을 경쟁자가 모두 쓰면 이득이 0으로 수렴 → 고유한 소스 기반 사실이 지속 가능한 차별점.
- **[논문]** E-GEO(이커머스 테스트베드): 효과적인 리스팅 리라이트는 "구체적 속성 + 혜택 프레이밍 + 질의 정렬 언어"라는 하나의 패턴으로 수렴.
- **[논문]** RAG 인용 연구: 문장의 **사실 정확성과 자립성(맥락 없이 인용 가능)**이 인용을 결정.

**개발 적용(반영 완료)**: 이 원칙들은 `geo-research_v1.md` §3.5–3.7로 RAG 코퍼스에 편입되어 policy checklist로 강제되고 있고, `geo-citation-content-agent`에는 verbatim 인용문/통계 하이라이트/신뢰도 연동 표현이 구현되어 있습니다.

## 4. CEP 기반 질의 커버리지 — 답변 모양의 콘텐츠

- **[공식]** Google AI Mode는 query fan-out(하나의 질문을 다수 하위 질의로 분해)으로 검색하므로, 하나의 일반 요약보다 **구매 상황별 하위 인텐트**(성분·효능·사용법·리뷰·비교·안전·변형·구매맥락)를 각각 답하는 콘텐츠 유닛이 유리합니다.
- **[산업]** ChatGPT 검색 유발 상위 용어: "reviews", "2025", "free", "features", "comparison" — 비교/리뷰형 프레이밍이 인용을 끌어옵니다.
- CEP(Category Entry Points, Ehrenberg-Bass)가 GEO 실무에 수입된 이유: 사용자 프롬프트 자체가 CEP("레티놀 후 민감해진 피부 장벽 크림")이므로, CEP 열거 → CEP별 콘텐츠 → CEP별 가시성 추적이 곧 GEO 전략입니다.

**개발 적용**: 이미 `cep_v1.md` + FAQ 재구성 파이프라인이 구현되어 있습니다. 확장 포인트: `inferredSearchQueries` 진단을 인용률 측정 도구의 프롬프트 패널로 연결하면 "생성한 CEP = 측정하는 프롬프트"의 폐루프가 됩니다.

## 5. 오프사이트 존재감 — 인용의 2/3는 소수 도메인에 집중

- **[산업]** 주제별로 약 30개 도메인이 인용의 약 2/3를 차지합니다. Reddit은 Google AI Overviews/Perplexity 최다 인용 도메인이었고(2024-08~2025-06), Wikipedia+Reddit이 미국 ChatGPT 인용의 25%+를 차지한 시기도 있습니다. YouTube 언급은 브랜드 가시성과 최고 상관(r≈0.74).
- **[산업]** Ahrefs 75K 브랜드 연구: **브랜드 웹 언급(r=0.664)이 백링크(r=0.218)보다 훨씬 강한 예측 변수** — 오프페이지 브랜드 풋프린트가 링크보다 중요합니다.
- 단, 도메인 점유율은 변동성이 큽니다(Reddit의 ChatGPT 인용 점유율이 2주 만에 ~60%→~10% 스윙한 사례). 단일 채널 의존은 위험합니다.

**개발 적용(반영 완료 + 확장)**: `geo-citation-content-agent`가 Reddit 채널을 담당합니다. surfaces 레지스트리가 확장 가능하므로 YouTube 스크립트/블로그(리스티클·비교형) surface 추가가 다음 단계 후보입니다. "Best X" 리스티클형은 ChatGPT 인용 페이지 유형의 ~44%를 차지한 기간이 있습니다.

## 6. 신선도 — 날짜는 신호다

- **[산업]** Perplexity는 신선하고 직접 답하는 페이지를 강하게 선호하고, 날짜 표기·최근 갱신 콘텐츠가 과대표집됩니다.
- **[공식]** IndexNow(Bing), 피드 15분 갱신(OpenAI)이 공식 신선도 채널입니다.

**개발 적용(부분 반영)**: 생성 파이프라인의 `freshnessStatement`/`observedAt`이 이미 신선도를 표기합니다. 확장: PDP 갱신 시 IndexNow ping을 보내는 훅.

## 7. 하지 말 것

- **llms.txt** — Google이 공식적으로 미사용 선언, 13.7만 사이트 조사에서 97%가 AI 봇 요청 0건. 투자 금지. **[공식]/[산업]**
- 키워드 스터핑, 반복 브랜드 언급, AI 전용 마크업 트릭, 인위적 질의 변형 페이지 — 역효과이거나 무효. **[논문]/[공식]**
- 단발 측정으로 효과 판단 — 생성 답변은 비결정적이라 동일 프롬프트도 매번 다른 소스를 인용합니다. 반드시 반복 샘플링. **[논문]**

## 8. 측정 — 효과 확인 채널

| 채널 | 성격 | 상태 |
| --- | --- | --- |
| 프롬프트 패널 반복 샘플링 (ChatGPT web_search / Perplexity citations / Gemini grounding API) | DIY, 우회적 | 내부 도구로 구현 가능 (measurement-guide 2-A) |
| Bing Webmaster Tools AI Performance | 공식 (Copilot/Bing 인용 데이터) | 도메인 소유 인증 후 사용 |
| 리퍼럴 트래픽 분석 (`utm_source=chatgpt.com`, perplexity.ai, gemini.google.com) | 공식 지표에 준함 | GA/로그에서 세그먼트 |
| 서버 로그 AI 봇 히트 (OAI-SearchBot 등) | 선행 지표 | 로그 파이프라인 필요 |
| 상용 도구 (Profound, Peec AI, Otterly, Semrush AI Toolkit, HubSpot AEO Grader) | 유료/무료 혼합 | 필요시 벤치마크 |

DIY 측정 주의: API 응답은 실제 소비자 UI 결과와 다를 수 있고, 각 provider ToS를 확인해야 하며, 지표는 절대값보다 **추세와 경쟁사 대비 SOV**로 해석해야 합니다.

## 9. 우선순위 요약 (근거 강도 × 구현 비용)

1. **봇 접근성 감사** — 비용 최소, 전제 조건 [공식]
2. **OpenAI 상품 피드 export** — 상품 질의 직결 [공식]
3. **통계/인용문/출처가 있는 보이는 본문** — 반영 완료, 유지 [논문]
4. **CEP 하위 인텐트 커버리지 + 측정 폐루프** — 콘텐츠 반영 완료, 측정 도구 연동은 후보 [공식]/[산업]
5. **Reddit/YouTube/리스티클 오프사이트 확장** — Reddit 반영 완료, YouTube/블로그 surface 후보 [산업]
6. **신선도 신호 자동화 (IndexNow 훅)** — 소규모 [공식]/[산업]
